Question 1: A financial company notices repeated login attempts from unusual locations every 39 seconds. What is the most likely cause?

  • A. Random user errors
  • B. Automated bot attacks

  • D. Routine software updates

Correct Answer: B

Reason: The fixed, repetitive timing of 39 seconds is a clear signature of an automated script or "bot" designed to perform brute-force attacks at regular intervals.

Lecture 1

Question 1: Theoretical Foundations

What are the core contributions of the Turing Machine and the Von Neumann Model to modern computing?

Answer:
The Turing Machine established the theoretical model of a universal programmable device that accepts input, processes it via instructions, and produces output. The Von Neumann Model added the practical architecture used today, proposing that both data and programs should be stored in the same Memory. It defined the four main hardware subsystems: Memory, Arithmetic Logic Unit (ALU), Control Unit (CU), and Input/Output.

Question 2: Memory and Storage Differences

Briefly distinguish between RAM vs. ROM and Magnetic Hard Drives vs. SSDs.

Answer:
  • Ram vs. ROM: RAM the memory that holds data and program instructions as the CPU works with them. RAM is volatile, meaning it holds data only when the power is on. When the power is off, RAM’s contents are lost. ROM : It is the memory that holds the Basic Input Output System (BIOS) of the computer, which is responsible of testing all the PC hardware as soon as it is turned on.
  • HDD vs. SSD: Magnetic Hard Drives (HDD) use moving mechanical parts to read magnetic disks, making them slower. Solid State Drives (SSD) use flash memory with no moving parts, making them significantly faster, more durable, and more efficient.

Question 3: OS Processing Techniques

How does an Operating System use Multitasking, Multithreading, and Multiprocessing to improve performance?

Answer:
  • Multitasking: The OS rotates rapidly between different programs on a single core to simulate simultaneous execution.
  • Multithreading: The OS rotates between different instructions (threads) within the same program to speed up specific tasks.
  • Multiprocessing: The OS utilizes multiple cores or CPUs to execute different programs truly simultaneously, rather than sequentially.

Question 4: Network Types and Structures

Summarize the differences between LAN and WAN, and compare Client/Server networks to Peer-to-Peer networks.

Answer:
  • LAN vs. WAN: A LAN (Local Area Network) connects computers in a small physical area using cables or Wi-Fi. A WAN (Wide Area Network) connects multiple LANs across large geographical distances using routers and leased lines.
  • Structure: In Client/Server networks, a central server manages security and resources while clients share the workload. In Peer-to-Peer (P2P) networks, all computers are equal, sharing files and devices directly with one another without a central server.

Lecture 2

Question 1: The CIA Triad

Explain the three core objectives of cybersecurity known as the CIA Triad. Provide a brief definition of each component and explain why all three are necessary for comprehensive information security.

Answer:
The CIA Triad consists of Confidentiality, Integrity, and Availability. Confidentiality ensures that private or confidential information is not disclosed to unauthorized individuals and that individuals control what information about them is collected and shared. Integrity ensures that information and programs are changed only in authorized ways (data integrity) and that systems perform their intended functions without manipulation (system integrity). Availability ensures that systems work promptly and services are not denied to authorized users. All three are essential because protecting only one or two aspects leaves systems vulnerable to different types of attacks and compromises overall security.

Question 2: Passive vs. Active Attacks

Differentiate between passive and active security attacks. Provide examples of each type and explain which one is typically more difficult to detect and why.

Answer:
Passive attacks attempt to learn or make use of information from a system without affecting system resources. Examples include eavesdropping and traffic analysis. These attacks do not modify data or disrupt operations.
Active attacks attempt to alter system resources or affect their operation. Examples include masquerade (impersonation), replay, modification of messages, and denial of service.
Passive attacks are typically more difficult to detect because they do not alter data or disrupt operations, leaving no obvious traces. Since the attacker only monitors communication without making changes, the system functions normally, making detection challenging without specialized monitoring tools.

Question 4: Cryptographic Algorithms

Describe the three main categories of cryptographic algorithms: keyless, single-key, and two-key algorithms.

Answer:
Keyless algorithms do not use keys and include cryptographic hash functions that create fixed-length digests. Use case: verifying data integrity or creating message authentication codes.
Single-key (symmetric) algorithms use one secret key shared between parties for both encryption and decryption. Use case: encrypting large amounts of data quickly
Two-key (asymmetric) algorithms use a private key (kept secret) and public key (shared openly). Use case: secure key exchange, digital signatures, or encrypting

Question 3: Explain what a Denial of Service (DoS) attack is and describe its primary impact on targeted systems

Answer:
A Denial of Service (DoS) attack targets servers or online services by overwhelming them with automated requests, making them slow, unstable, or completely unavailable to allowed users. primary impact is service disruption; legitimate users cannot access websites or applications, potentially causing business losses and reputation damage.

Question 4: List at least two symptoms that indicate a system might be under a DoS attack.

Answer:
(1) inability to access a website or service
(2) extremely slow response times or delays in loading content
(3) increased spam emails combined with overall poor system performance.

Question 5: Describe three different mitigation techniques that can protect systems from DoS attacks. For each technique describe it shortly

Answer:
Traffic routing and filtering: routes suspicious traffic to scrubbing centers where rate-limiting and CAPTCHA verification filter out malicious requests while allowing legitimate users through.
Load balancers: These distribute incoming traffic across multiple servers, detecting high CPU usage and automatically redirecting connections to underutilized nodes.
Intrusion Detection and Prevention Systems (IDS/IPS): IDS detects attacks by monitoring for excessive login attempts or packet floods, alerting security teams for response, while IPS actively blocks malicious traffic by dropping packets and temporarily blocking attacker IP ranges automatically.

Question 7: Compare between ping flood and syn flood attack

Answer:
SYN Flood attacks exploit the TCP three-way handshake by sending numerous SYN packets but never completing the connection with the final ACK. This causes the server to keep half-open connections waiting. ICMP Flooding attacks use ping requests (ICMP Echo Request packets) sent in massive quantities to overwhelm the target. The command ping -f sends packets. Both attacks consume bandwidth and CPU resources.

Question 1: Define a Man-in-the-Middle (MITM) attack

Answer:
A Man-in-the-Middle (MITM) attack occurs when a hacker secretly intercepts and potentially modifies communication between a user and a legitimate server.

Question 2: What is the impact of MIM attack

Answer:

  2. Financial loss
  3. Account compromise/ information breach

Question 5: Explain how Cross-Site Scripting (XSS) functions as a type of MITM attack. Describe the mechanism by which malicious scripts are injected into trusted websites and explain what specific types of data these scripts typically steal.

Answer:
Cross-Site Scripting (XSS) functions as a MITM attack by injecting malicious JavaScript or other scripting code into trusted, websites. When victims visit these infected sites, their browsers automatically execute malicious scripts.

Question 6: Mention two of most common XSS attacks?

Answer:
  1. Reflected XSS
  2. Stored XSS
  3. Dom XSS

Question 7: Describe how DNS spoofing or HTTP spoofing works as a MITM attack technique. Explain what makes this attack particularly deceptive for victims and why URLs may appear correct even when connecting to malicious servers.

Answer:
DNS spoofing or HTTP spoofing works by redirecting victims to fake websites while making them believe they are accessing legitimate servers. The attacker poisons DNS caches or manipulates DNS responses, causing the victim computer to receive incorrect IP addresses for trusted domains.

Question 8: Describe how email hijacking works as a MITM attack and explain how attackers gain access to victim email accounts.

Answer:
Email hijacking as a MITM attack occurs when attackers gain unauthorized access to a victim email account, allowing them to intercept, monitor, and manipulate all communications flowing through that account.
