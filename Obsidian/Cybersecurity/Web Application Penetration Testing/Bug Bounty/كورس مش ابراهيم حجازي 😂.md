- What is WebDav?
  [WebDAV - Wikipedia](https://en.wikipedia.org/wiki/WebDAV)
  - Exploiting WebDAV (Web Distributed Authoring and Versioning) in web penetration testing involves taking advantage of vulnerabilities in the WebDAV service to gain unauthorized access or manipulate files on a web server](https://www.infosecinstitute.com/resources/penetration-testing/learning-pentesting-metasploitable3-exploiting-webdav/). Here are some common steps and methods:

 1. **Information Gathering**: Use tools like Nmap to identify if WebDAV is enabled on the target server and gather details about the server configuration](https://www.infosecinstitute.com/resources/penetration-testing/learning-pentesting-metasploitable3-exploiting-webdav/).
 2. **Authentication**: Often, WebDAV requires authentication. You may need to use brute force attacks or exploit weak credentials to gain access.
 3. **File Upload**: If WebDAV is misconfigured, you might be able to upload files to the server](https://www.infosecinstitute.com/resources/penetration-testing/learning-pentesting-metasploitable3-exploiting-webdav/). This can be done using tools like `curl` or `cadaver`](https://www.infosecinstitute.com/resources/penetration-testing/learning-pentesting-metasploitable3-exploiting-webdav/).
 4. **Execute Code**: Upload a webshell (e.g., a PHP script) to the server](https://www.infosecinstitute.com/resources/penetration-testing/learning-pentesting-metasploitable3-exploiting-webdav/). This script can be used to execute commands on the server](https://www.infosecinstitute.com/resources/penetration-testing/learning-pentesting-metasploitable3-exploiting-webdav/).
 5. **Post-Exploitation**: Once you have access, you can perform further actions such as escalating privileges, extracting data, or pivoting to other systems.

		Here's an example using `curl` to upload a file:
		
		```sh
			curl -X PUT --upload-file shell.php http://<target_ip>/webdav/uploads/shell.php
		```
		
		And using `cadaver` to interact with the WebDAV server:
		
		```sh
		cadaver http://<target_ip>/webdav
		put shell.php
		```
		
		Always ensure you have permission to test and exploit WebDAV services, as unauthorized access is illegal and unethical.
