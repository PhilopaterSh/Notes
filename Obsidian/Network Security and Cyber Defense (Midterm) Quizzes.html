Network Security (Midterm) Quizzes

Quiz 1 – Networking Fundamentals (OSI Model & Devices)

1. What does OSI stand for?

  • Open Security Infrastructure
  • Operating Signal Interface
  • Open Systems Interconnection
  • Operational System Interface

Correct Answer: c) Open Systems Interconnection

Reason: OSI stands for Open Systems Interconnection, a conceptual framework used to describe the functions of a networking system.

2. How many layers are in the OSI model?

  • 8
  • 6
  • 5
  • 7

Correct Answer: d) 7

Reason: The OSI model consists of seven distinct layers: Physical, Data Link, Network, Transport, Session, Presentation, and Application.

3. Which layer of the OSI model is responsible for IP addressing?

  • Network
  • Data Link
  • Transport
  • Physical

Correct Answer: a) Network

Reason: The Network layer (Layer 3) is responsible for logical addressing (IP addresses) and routing packets across different networks.

4. A switch operates mainly at which OSI layer?

  • Layer 2
  • Layer 4
  • Layer 1
  • Layer 7

Correct Answer: a) Layer 2

Reason: A network switch operates primarily at Layer 2 (Data Link Layer) of the OSI model, forwarding data frames based on MAC addresses.

5. A router operates mainly at which OSI layer?

  • Layer 3
  • Layer 2
  • Layer 1
  • Layer 7

Correct Answer: a) Layer 3

Reason: A router operates at Layer 3 (Network Layer) of the OSI model, responsible for routing packets between different networks using IP addresses.

6. What is the main function of a switch?

  • Connect networks
  • Forward packets based on IP
  • Forward frames based on MAC addresses
  • Translate domain names

Correct Answer: c) Forward frames based on MAC addresses

Reason: A switch's primary function is to forward data frames to specific devices on a local network using their MAC addresses.

7. What is the main function of a router?

  • Filter traffic based on port numbers
  • Route packets between networks
  • Create VLANs
  • Generate MAC addresses

Correct Answer: b) Route packets between networks

Reason: A router's main function is to connect different networks and route data packets between them, determining the best path for data delivery.

8. Which OSI layer is responsible for end-to-end reliable communication?

  • Application
  • Transport
  • Network
  • Session

Correct Answer: b) Transport

Reason: The Transport layer (Layer 4) is responsible for end-to-end communication, including reliability, flow control, and segmentation/reassembly of data.

9. Which device reduces collision domains but not broadcast domains?

  • Router
  • Hub
  • Switch
  • Firewall

Correct Answer: c) Switch

Reason: A switch creates separate collision domains for each port, thus reducing collisions. However, it operates within a single broadcast domain by default.

10. In security context, why is understanding OSI layers important?

  • It allows precise identification of attack points and defense mechanisms.
  • It reduces the need for encryption.
  • It helps choose cheaper hardware.
  • It increases network speed.

Correct Answer: a) It allows precise identification of attack points and defense mechanisms.

Reason: Understanding the OSI layers helps security professionals pinpoint where vulnerabilities might exist and implement appropriate security measures at each layer.

Quiz 2 – IP Addressing, Subnetting & Network Devices

1. What is the default subnet mask for a Class C IPv4 network?

  • 255.0.0.0
  • 255.255.0.0
  • 255.255.255.0
  • 255.255.255.255

Correct Answer: c) 255.255.255.0

Reason: Class C networks are designed for smaller networks and have a default subnet mask of 255.255.255.0, indicating the first three octets are for the network portion.

2. Which IP address type is used for one-to-many communication?

  • Unicast
  • Broadcast
  • Multicast
  • Anycast

Correct Answer: c) Multicast

Reason: Multicast IP addresses are used for one-to-many communication, where data is sent to a group of interested recipients simultaneously.

3. What does NAT stand for?

  • Network Address Translation
  • Network Access Type
  • Node Allocation Table
  • Network Application Transport

Correct Answer: a) Network Address Translation

Reason: NAT (Network Address Translation) is a method of remapping one IP address space into another by modifying network address information in the IP header of packets.

4. How many usable host addresses are there in a /30 subnet?

  • 2
  • 4
  • 6
  • 8

Correct Answer: a) 2

Reason: A /30 subnet uses 30 bits for the network portion, leaving 2 bits for hosts. This results in 2^2 = 4 total addresses. Subtracting the network address and broadcast address leaves 2 usable host addresses.

5. Which device operates primarily to connect different networks?

  • Switch
  • Router
  • Hub
  • Access Point

Correct Answer: b) Router

Reason: A router's primary function is to connect distinct networks and facilitate communication between them by forwarding packets.

6. Which command shows the IP configuration on a Windows PC?

  • ifconfig
  • ipconfig
  • netstat
  • ping

Correct Answer: b) ipconfig

Reason: On Windows systems, `ipconfig` is used to display current TCP/IP network configuration values, while `ifconfig` is common on Unix-like systems.

7. What is the purpose of a VLAN?

  • Extend a network over the internet
  • Isolate broadcast domains within a switch
  • Increase IP addresses
  • Encrypt data between devices

Correct Answer: b) Isolate broadcast domains within a switch

Reason: VLANs (Virtual Local Area Networks) logically segment a network into separate broadcast domains, even if devices are on the same physical switch.

8. Which IP address is considered a private IP address?

  • 8.8.8.8
  • 192.168.1.1
  • 172.217.3.110
  • 54.239.28.85

Correct Answer: b) 192.168.1.1

Reason: 192.168.1.1 falls within the private IP address range (192.168.0.0 to 192.168.255.255), which are not routable on the public internet.

9. What is the purpose of a subnet mask?

  • To encrypt network traffic
  • To divide an IP network into smaller networks
  • To identify the router’s MAC address
  • To assign DNS servers

Correct Answer: b) To divide an IP network into smaller networks

Reason: A subnet mask is used to distinguish the network portion of an IP address from the host portion, allowing for the division of a larger network into smaller subnets.

10. Which command can test connectivity between two devices on a network?

  • ping
  • traceroute
  • nslookup
  • arp

Correct Answer: a) ping

Reason: The `ping` command sends ICMP echo request packets to a target host and listens for echo reply packets, effectively testing basic network connectivity.

Quiz 1: Routing & Routing Table Fundamentals

Q1. What is the primary purpose of a routing table in a router?

  • To store MAC addresses of connected devices
  • To determine the best path for packet forwarding
  • To encrypt network traffic
  • To manage application-layer services

Correct Answer: B

Reason: A routing table stores information about network destinations and the "next hop" to reach them, enabling a router to determine the optimal path for forwarding data packets.

Q2. Which of the following entries is typically found in a routing table?

  • A. DNS server address
  • B. Destination network
  • C. Application port number
  • D. User credentials

Correct Answer: B

Reason: Routing tables primarily contain entries for destination networks, indicating which network can be reached and how.

Q3. Which metric represents the number of routers a packet must pass through?

  • A. Bandwidth
  • B. Delay
  • C. Hop count
  • D. Reliability

Correct Answer: C

Reason: Hop count is a routing metric that measures the number of routers (hops) a packet traverses from its source to its destination.

Q4. Administrative Distance (AD) is used to:

  • A. Measure link speed
  • B. Select the best routing protocol source
  • C. Encrypt routing updates
  • D. Detect routing loops

Correct Answer: B

Reason: Administrative Distance is a value used by routers to rank the trustworthiness of routing information sources, with lower AD values indicating more preferred sources when multiple paths to a destination exist.

Q5. If multiple routes exist to the same destination, the router chooses the route with:

  • A. Highest hop count
  • B. Lowest metric value
  • C. Highest IP address
  • D. Largest routing table

Correct Answer: B

Reason: Routers typically choose the path with the lowest metric value (e.g., lowest hop count, highest bandwidth) when multiple routes to the same destination are available, after considering administrative distance.

Q6. Which command displays the routing table on a Cisco router?

  • A. show cdp neighbors
  • B. show ip route
  • C. show interfaces
  • D. show protocols

Correct Answer: B

Reason: The `show ip route` command is used on Cisco IOS devices to display the IP routing table and its contents.

Q7. What does the “next hop” field in a routing table indicate?

  • A. Final destination host
  • B. Router’s own IP address
  • C. Next router to forward the packet to
  • D. Default gateway only

Correct Answer: C

Reason: The "next hop" in a routing table specifies the IP address of the adjacent router to which a packet should be forwarded on its way to the ultimate destination network.

Q8. Which routing table entry is used when no specific route matches a destination?

  • A. Static route
  • B. Direct route
  • C. Default route
  • D. Host route

Correct Answer: C

Reason: A default route is a special type of static route used by a router when it does not have a specific entry in its routing table for a destination network, essentially sending all unmatched traffic to a single exit point.

Q9. Which routing table component specifies where the packet leaves the router?

  • A. Metric
  • B. Destination network
  • C. Outgoing interface
  • D. Administrative distance

Correct Answer: C

Reason: The outgoing interface in a routing table specifies the local interface on the router through which the packet should be sent to reach the next hop or destination network.

Q10. Which layer of the OSI/TCP-IP model is routing mainly associated with?

  • A. Application layer
  • B. Transport layer
  • C. Network layer
  • D. Data link layer

Correct Answer: C

Reason: Routing is a fundamental function of the Network layer (Layer 3) in both the OSI and TCP/IP models, as it involves determining the best paths for data packets across interconnected networks.

Quiz 2: Routing Protocols & Network Concepts

Q1. RIP is classified as which type of routing protocol?

  • A. Link State
  • B. Path Vector
  • C. Distance Vector
  • D. Hybrid

Correct Answer: C

Reason: RIP (Routing Information Protocol) is a distance-vector routing protocol that uses hop count as its metric to determine the best path to a destination.

Q2. What metric does RIP use to select the best path?

  • A. Bandwidth
  • B. Delay
  • C. Cost
  • D. Hop count

Correct Answer: D

Reason: RIP uses hop count, the number of routers a packet must cross, as its sole metric for determining the best path.

Q3. What is the maximum hop count allowed in RIP?

  • A. 10
  • B. 15
  • C. 16
  • D. 255

Correct Answer: B

Reason: RIP has a maximum hop count of 15, meaning any destination beyond 15 hops is considered unreachable.

Q4. Why is RIP considered slow to converge?

  • A. It uses Dijkstra’s algorithm
  • B. It sends updates every 30 seconds
  • C. It uses LSAs
  • D. It supports multiple areas

Correct Answer: B

Reason: RIP is considered slow to converge because it sends full routing table updates periodically (every 30 seconds), which can lead to delays in propagating network changes.

Q5. OSPF calculates the shortest path using which algorithm?

  • A. Bellman-Ford
  • B. DUAL
  • C. SPF (Dijkstra)
  • D. AS-Path

Correct Answer: C

Reason: OSPF (Open Shortest Path First) is a link-state routing protocol that uses Dijkstra's algorithm (Shortest Path First - SPF) to calculate the shortest path to all destinations.

Q6. What is the backbone area in OSPF called?

  • A. Area 1
  • B. Area 10
  • C. Area 100
  • D. Area 0

Correct Answer: D

Reason: In OSPF, Area 0 (or Area 0.0.0.0) is designated as the backbone area, through which all other non-backbone areas must connect.

Q7. Which routing protocol supports unequal-cost load balancing?

  • A. RIP
  • B. OSPF
  • C. EIGRP
  • D. BGP

Correct Answer: C

Reason: EIGRP (Enhanced Interior Gateway Routing Protocol) is known for its ability to perform unequal-cost load balancing, allowing traffic to be distributed across paths with different metrics.

Q8. Which protocol is primarily used to connect different Autonomous Systems (AS)?

  • A. OSPF
  • B. RIP
  • C. EIGRP
  • D. BGP

Correct Answer: D

Reason: BGP (Border Gateway Protocol) is an exterior gateway protocol primarily used to exchange routing information between different Autonomous Systems on the internet.

Q9. Which routing protocol prioritizes policy and stability over fast convergence?

  • A. RIP
  • B. OSPF
  • C. EIGRP
  • D. BGP

Correct Answer: D

Reason: BGP (Border Gateway Protocol) prioritizes policy-based routing and network stability over rapid convergence, as it manages routing across large, complex internetworks.

Q10. Which protocol is known as the “error reporting and diagnostic protocol” of the Internet?

  • A. ARP
  • B. ICMP
  • C. TCP
  • D. UDP

Correct Answer: B

Reason: ICMP (Internet Control Message Protocol) is used by network devices, including routers, to send error messages and operational information, making it the "error reporting and diagnostic protocol."

Quiz 1: The Security Onion & Ethical Hacking

  • A. Perimeter Security
  • B. Data Security
  • C. Network Security
  • D. Policies and Awareness

Correct Answer: B

Reason: In the "Security Onion" model, data security is typically the innermost layer, as the ultimate goal of all other security measures is to protect the sensitive data.

  • B. Gaining Access

  • A. Nmap
  • B. IDS/IPS
  • C. Hashcat
  • D. Metasploit

Reason: IDS (Intrusion Detection Systems) and IPS (Intrusion Prevention Systems) are used within the internal network to monitor for and react to suspicious activity, including lateral movement.

  • B. Scope Limitations

  • B. To bypass the Firewall

  • B. Policies, Procedures, and Awareness
  • C. Perimeter Security
  • D. Data Security

Reason: Social engineering attacks primarily target the human element, making "Policies, Procedures, and Awareness" the layer most relevant to defending against such attacks through education and training.

Q10. Which document is essential for a White Hat hacker to avoid being treated as a criminal?

Quiz 1: Packet Reception and Processing

Q1. What is the primary role of a Network Interface Card (NIC)?

  • A. To provide Internet access via web browsers
  • B. To link a machine to a network and handle frame reception
  • C. To execute user applications
  • D. To encrypt packets

Correct Answer: B

Reason: A NIC's fundamental role is to enable a device to connect to a network medium, sending and receiving data frames.

Q2. Which of the following is true about Ethernet and WiFi networks?

  • A. They only send data to specific devices
  • B. They are broadcast media, and all devices can hear transmitted frames
  • C. They use NICs that ignore broadcast frames
  • D. They do not require MAC addresses

Correct Answer: B

Reason: Both Ethernet and Wi-Fi are broadcast media by nature, meaning that all devices connected to the same segment or access point can potentially "hear" all transmitted frames, though filtering mechanisms exist.

Q3. When a frame arrives at a NIC, what is the first step?

  • A. The CPU immediately processes it
  • B. The frame is copied into the NIC’s internal memory
  • C. The packet is discarded
  • D. The protocol stack handles it directly

Correct Answer: B

Reason: Upon reception, a frame is typically first copied into the NIC's internal buffer or memory for initial processing before being passed to the system's main memory.

Q4. What mechanism allows a NIC to transfer packets to system memory without involving the CPU?

  • A. Interrupt-driven I/O
  • B. Direct Memory Access (DMA)
  • C. Kernel queue
  • D. Promiscuous mode

Correct Answer: B

Reason: Direct Memory Access (DMA) allows the NIC to directly transfer data to and from the system's main memory without requiring constant intervention from the CPU, improving efficiency.

Q5. After a NIC interrupts the CPU, what does the kernel do with incoming packets?

  • A. Discards them immediately
  • B. Moves packets from the ring buffer to a kernel queue for processing
  • C. Sends them directly to user-space applications
  • D. Converts them to TCP packets only

Correct Answer: B

Reason: After the NIC's DMA engine has placed the incoming packet data into a ring buffer in system memory, the NIC interrupts the CPU. The kernel's network driver then moves these packets from the ring buffer into kernel queues for further processing by the protocol stack.

Q6. Which layer removes link-layer headers such as Ethernet before passing packets upward?

  • A. Application layer
  • B. Protocol stack
  • C. Link-layer driver
  • D. Raw socket interface

Correct Answer: C

Reason: The link-layer driver (part of the operating system's kernel) is responsible for handling the specifics of the data link layer, including removing the Ethernet (or Wi-Fi) header before passing the packet up to the network layer.

Q7. What is the purpose of the kernel’s protocol stack in packet reception?

  • A. To encrypt packets
  • B. To decapsulate packets and process protocol-specific headers
  • C. To block unwanted traffic
  • D. To transfer packets to other NICs

Correct Answer: B

Reason: The kernel's protocol stack (e.g., TCP/IP stack) decapsulates incoming packets layer by layer, processing headers relevant to each protocol (IP, TCP/UDP) and eventually delivering the data to the appropriate application.

Q8. At the hardware level, what does the NIC controller do?

  • A. Generates MAC addresses
  • B. Manages and coordinates data movement inside the NIC
  • C. Passes packets directly to applications
  • D. Runs the kernel network stack

Correct Answer: B

Reason: The NIC controller is the microchip on the NIC that handles the physical and data link layer functions, including managing the flow of data between the network medium and the system's bus/memory.

Q9. Where are incoming packets temporarily stored in the system memory before kernel processing?

  • A. Stack buffer
  • B. Ring buffer
  • C. Promiscuous queue
  • D. User-space application buffer

Correct Answer: B

Reason: Incoming packets are often temporarily stored in a "ring buffer" (a circular buffer) in the system's memory by the NIC's DMA mechanism before the kernel's network stack can process them.

Q10. Which of the following represents the two levels of packet reception explained in the text?

  • A. Hardware level (NIC) and System level (Kernel)
  • B. Application layer and Transport layer
  • C. TCP/IP stack and UDP stack
  • D. User-space and Network driver only

Correct Answer: A

Reason: Packet reception is typically understood at two main levels: the hardware level (handled by the NIC) and the software/system level (handled by the kernel and its network stack).

Quiz 2: Modes and Packet Sniffing

Q1. What is promiscuous mode used for in wired networks?

  • A. To block all packets except broadcast
  • B. To forward every frame to the kernel regardless of the MAC address
  • C. To encrypt data on the network
  • D. To reduce network traffic

Correct Answer: B

Reason: In promiscuous mode, a NIC is configured to capture and process all network traffic it receives, regardless of whether the frames are addressed to its own MAC address or not.

Q2. Which privilege is typically required to enable promiscuous mode?

  • A. Guest user access
  • B. Administrator or root access
  • C. Any user-level access
  • D. No privilege required

Correct Answer: B

Reason: Enabling promiscuous mode often requires elevated privileges (administrator or root) on most operating systems, as it allows access to network traffic not specifically destined for the host.

Q3. Why is promiscuous mode alone insufficient for wireless networks?

  • A. Wireless NICs cannot read MAC addresses
  • B. Wireless networks operate on multiple channels and bandwidth slices
  • C. Wireless NICs do not use protocol stacks
  • D. Wireless traffic cannot be captured at all

Correct Answer: B

Reason: Wireless networks operate using different channels and can employ techniques like channel hopping or multiple spatial streams. A wireless NIC in promiscuous mode on a single channel might miss traffic on other channels or streams, making it less effective for comprehensive sniffing without further configuration (e.g., monitor mode).

Q4. What is the purpose of the BSD Packet Filter (BPF)?

  • A. To encrypt packets before sending
  • B. To filter packets inside the kernel before delivering to user-space
  • C. To bypass the NIC entirely
  • D. To change the MAC address of packets

Correct Answer: B

Reason: BPF (BSD Packet Filter) provides a mechanism to efficiently filter network packets at the kernel level, reducing the overhead of passing irrelevant packets to user-space applications.

Q5. How are BPF filters attached to a socket in Unix-based systems?

  • A. Using recvfrom()
  • B. Using setsockopt() with SO_ATTACH_FILTER
  • C. Using bind()
  • D. Using raw socket creation

Correct Answer: B

Reason: In Unix-based systems, BPF filters are typically attached to a socket using the `setsockopt()` function with the `SO_ATTACH_FILTER` option.

Q6. What advantage does BPF filtering provide for packet sniffing?

  • A. Ensures all packets are captured
  • B. Reduces CPU and memory usage by discarding irrelevant packets early
  • C. Encrypts captured packets automatically
  • D. Eliminates the need for NICs

Correct Answer: B

Reason: BPF filtering significantly improves packet sniffing performance by allowing the kernel to discard packets that don't match the filter criteria, thereby reducing the amount of data transferred to user-space and conserving CPU/memory resources.

Q7. Why are raw sockets required for packet sniffing?

  • A. To send packets faster
  • B. To capture all packets on the network regardless of destination IP or port
  • C. To convert UDP to TCP
  • D. To avoid using BPF

Correct Answer: B

Reason: Raw sockets allow an application to directly access the network layer, enabling it to send and receive raw IP packets and capture traffic without the operating system's protocol stack processing it based on destination IP or port.

Q8. Which socket type is used to create a raw socket?

  • A. SOCK_STREAM
  • B. SOCK_DGRAM
  • C. SOCK_RAW
  • D. SOCK_PROMISC

Correct Answer: C

Reason: The `SOCK_RAW` socket type is used to create raw sockets, providing direct access to the underlying network protocols.

Q9. What library is commonly used to optimize packet capturing and provide portability across systems?

  • A. libnet
  • B. libpcap
  • C. libsocket
  • D. libfilter

Correct Answer: B

Reason: libpcap is a widely used, portable C/C++ library for network traffic capture, providing a consistent API across various operating systems for tools like Wireshark and tcpdump.

Q10. In packet sniffing using pcap, what is the first step?

  • A. Compile a BPF filter
  • B. Open a live pcap session on a network interface
  • C. Set the filter using pcap_setfilter()
  • D. Typecast the Ethernet header

Correct Answer: B

Network Security Lecture [1] - Question 1

What is the fundamental purpose of a communication system, and what are its main components?

Answer:
The fundamental purpose of a communication system is the exchange of data between two or more devices.
Its main components are:
  1. Source/Information – The original message (data, voice, or video).
  2. Transmitter (Tx) – Converts the message into a suitable signal for transmission.
  3. Channel/Medium – The physical path the signal travels (cable, fiber, air).
  4. Receiver (Rx) – Converts the received signal back into the original message.
  5. Destination – The intended user or device.

Network Security Lecture [1] - Question 2

Explain the Data Communication Model from text transmission to reception.

Answer:
The Data Communication Model follows these steps:
  1. Input Information (m) – Original data such as text.
  2. Input Data (g(t)) – Conversion of data into a digital bit stream.
  3. Transmission Preparation – Modulation creates the transmitted signal (s(t)).
  4. Signal Transmission – The signal travels through the medium and may degrade.
  5. Signal Recovery – The receiver demodulates the signal back into bits.
  6. Output Data (g'(t)) – The destination converts bits back to the original data.

Network Security Lecture [1] - Question 5

How does a switch operate, and which OSI layer does it work on?

Answer:
A switch operates at Layer 2 (Data Link Layer) of the OSI model.
  • It learns MAC addresses by inspecting the source MAC of incoming frames.
  • Stores MAC addresses in a MAC address table with corresponding ports.
  • If the destination MAC is known, it forwards the frame to the correct port.
  • If unknown, it floods the frame to all ports except the source port.

Network Security Lecture [1] - Question 6

Explain NAT and mention its main types.

Answer:
Network Address Translation (NAT) allows multiple private devices to share a single public IP address to access the internet.
How it works:
  • Replaces private IP addresses with a public IP.
  • Uses port numbers to track sessions.
  • Maintains a NAT table for translations.
Types of NAT:
  1. Static NAT – One-to-one permanent mapping.
  2. Dynamic NAT – Temporary mapping from a pool of public IPs.
  3. PAT (Port Address Translation) – Many private IPs share one public IP using different ports.

Network Security Lecture [3] - Question 1

Explain how a Network Interface Card (NIC) receives a packet at the hardware level.

Answer:
At the hardware level, packet reception occurs as follows:
  1. The packet arrives from the network and enters the NIC through the Net Receive DMA engine.
  2. The packet is temporarily stored in the NIC's internal memory buffers.
  3. An on-board NIC controller manages the process.
  4. Using Direct Memory Access (DMA), the packet is transferred directly to the host's main memory (RAM) over the PCI bus.
DMA allows high-speed transfer without involving the CPU, improving performance.

Network Security Lecture [3] - Question 2

Describe the system-level packet processing flow inside the operating system kernel.

Answer:
After the packet reaches system memory:
  1. The packet is placed into a ring buffer in RAM.
  2. The NIC interrupts the CPU to signal packet arrival.
  3. The CPU moves packets from the ring buffer into a kernel queue.
  4. The kernel invokes protocol-specific handlers.
  5. Link-layer headers are removed, and protocols such as IP, TCP, or UDP are processed.
  6. The cleaned data is delivered to the appropriate user-space application.

Network Security Lecture [3] - Question 3

What is promiscuous mode, and why is it required for packet sniffing on wired networks?

Answer:
Promiscuous mode is a special NIC operating mode where:
  • The NIC forwards all received frames to the operating system.
  • Frames are forwarded regardless of destination MAC address.
It is required for packet sniffing because, under normal operation, a NIC discards frames not addressed to it. Promiscuous mode allows tools like Wireshark or tcpdump to capture traffic between other devices on the same network. Enabling it usually requires administrator or root privileges.

Network Security Lecture [3] - Question 4

Why is monitor mode needed for wireless packet sniffing instead of promiscuous mode?

Answer:
In wireless networks:
  • Traffic is distributed across multiple channels.
  • Wireless NICs normally listen only on specific channels.
Promiscuous mode alone is insufficient because it cannot capture traffic from other channels. Monitor mode allows a wireless NIC to:
  • Capture raw 802.11 frames
  • Listen to all traffic on a selected channel
  • Analyze management, control, and data frames
This makes monitor mode essential for wireless packet analysis.

Network Security Lecture [3] - Question 5

What is the BSD Packet Filter (BPF), and why is it important?

Answer:
The BSD Packet Filter (BPF) is a kernel-level packet filtering mechanism that:
  • Filters packets before they reach user-space applications
  • Allows only relevant packets to be delivered
  • Reduces CPU usage and memory overhead
BPF is important because capturing all packets and filtering them in user space is inefficient, especially on high-traffic networks.

Network Security Lecture [3] - Question 6

Explain how a BPF filter works from attachment to packet decision.

Answer:
  1. A user-space application attaches a BPF filter to a socket.
  2. The filter is written in human-readable form (e.g., tcp, port 22).
  3. The filter is compiled into BPF bytecode.
  4. The kernel executes this bytecode using the BPF pseudo-machine.
  5. If the packet matches the filter, it is accepted and delivered.
  6. If not, it is discarded early inside the kernel.
This ensures efficient packet processing.

Network Security Lecture [3] - Question 7

Why are raw sockets required for packet sniffing, and what are their key properties?

Answer:
Raw sockets are required because:
  • Normal sockets receive only packets addressed to their IP and port.
  • Packet sniffing requires capturing all packets, regardless of destination.
Key properties of raw sockets:
  • Created using SOCK_RAW
  • Receive entire packets including link-layer headers
  • Capture packets before protocol stack filtering
  • Copy packets without blocking or intercepting them

Network Security Lecture [3] - Question 8

Why is the pcap API preferred over simple raw socket sniffers?

Answer:
The pcap API is preferred because:
  • It is platform-independent
  • Offers high-performance packet capture
  • Supports human-readable filtering rules
  • Uses BPF for efficient kernel-level filtering
  • Reduces packet loss under heavy traffic
Simple raw socket sniffers suffer from poor performance, lack filtering, and may miss packets on busy networks.
